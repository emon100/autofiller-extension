Prompt 主题：开发浏览器扩展的“保存/学习”交互通知气泡组件

任务目标： 基于 React、Tailwind CSS 和 framer-motion（用于动画），在浏览器扩展的 Content Script 中实现一套浮动控制挂件（Widget）及其触发的“学习反馈与确认”交互流程。请复刻参考图中的三步 UI 状态和指定的交互动效。

核心要求：

整个交互必须具有高度的响应感（Reactive feel）和流畅的过渡动画（Animations），通过 framer-motion 实现气泡的弹出、切换和消失。
界面分为三个关键状态：初始挂件状态、学习反馈气泡状态、保存成功的确认气泡状态。
详细实现步骤与 UI 规范：

阶段一：初始浮动挂件 (Initial Floating Widget)
位置： 页面右下角（或可拖动位置），覆盖在宿主页面之上。
UI 元素：
一个紧凑的容器，包含两个主按钮：“Save”（保存）和“Fill”（填充）。
容器看起来应该是一个轻量级、可交互的工具栏（参考图中的圆角矩形和拖拽手柄暗示）。
在主容器下方，有一个小字号链接：“Manage Database”（管理信息库）。
交互动作： 点击“Save”按钮触发阶段二。
阶段二：学习反馈气泡 (Learning Feedback Popover)
触发机制： 点击“Save”后，模拟一个短暂的表单解析过程，然后以动画形式弹出此气泡。
样式：
一个较大的对话气泡（Speech bubble），带有指向刚才点击的“Save”按钮的尖角。
背景色为淡蓝色（参考图）。
内容元素：
标题： “我刚刚学到了这些：”
副标题/提示： “（表格里面的每一行都是可以删除可以编辑）”
数据编辑区： 一个列表区域，展示模拟捕获的字段数据。每一行包含：
一个绿色的勾选确认图标（或复选框）。
一个红色的“X”删除图标。
可编辑的文本（例如：“性别：trans”）。 注：AI实现时请提供mock数据展示该效果，且确保这些字段是 Input 框 enable 了编辑功能。
操作按钮： 底部有一个圆形的“ok”按钮。
交互动效： 气泡应带有弹性和渐显效果。数据行的删除操作应有对应的移除动画。点击“ok”触发阶段三。
阶段三：保存成功确认气泡 (Confirmation Popover)
触发机制： 点击阶段二的“ok”按钮后，气泡内容平滑过渡到此状态。
样式： 保持同样的对话气泡容器和指向角，但内容变化。
内容元素：
图标： 一个巨大的绿色对勾图标居中显示。
成功文本： “已经保存到信息库。”
跳转链接： 在文本下方提供一个链接：“view/edit 信息库”，点击该链接应触发打开新页面（扩展管理页/Option页）的动作。
操作按钮： 底部同样有一个圆形的“ok”按钮。
交互动效：
从阶段二过渡到阶段三时，需要有一个平滑的内容切换动画（例如：旧内容淡出缩小，新内容淡入放大，同时那个绿色大对勾有一个强调的弹跳动画效果）。
点击“ok”按钮，气泡以收缩或淡出动画关闭，返回阶段一的状态。
技术栈建议：

React (用于组件构建)
Tailwind CSS (用于样式复刻，实现 shadcn/ui 风格的质感)
Framer Motion (用于实现所有要求的动画和响应感)
Radix UI Popover (可选，作为气泡组件的基础可访问性原语)
